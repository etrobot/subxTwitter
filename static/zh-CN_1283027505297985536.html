我刚刚发现我的Photo AI R2（@Cloudflare的S3）存储桶位于欧洲西部，因为当我在仪表板上创建它时，Cloudflare根据你当前的浏览器位置自动选择了它，而我当时在欧洲。

Aws\Result Object ( [data:Aws\Result:private] => Array ( [LocationConstraint] => WEUR

但我的Web服务器在美国，Replicate也在美国，所以它会进行很多不必要的美欧之间的往返。

有什么办法能够轻松地将Cloudflare R2存储桶迁移到新位置吗？我可以创建一个新的存储桶，但不确定如何搬移所有内容。

现在存储桶中有34.89TB的数据，太多了，搬迁也有些困难。


06 Jan 2024 15:37:06
作者：@levelsio
好了，搞定了，我刚创建了一个新的存储桶photoai-us-west，并将其与r2-us-west.photoai.com一起使用。

我发现美欧的传输需要从Replicate美到我的服务器美再到R2欧再到Replicate美等等，过程中会花费很长时间。

在Daniel Lockyer为我在服务器上设置New Relic之后，欧洲R2花费了1500ms（橙色），美国使用了500ms（浅粉色），所以现在快了3倍！

不只只是一次往返，因为整个后期处理管道都涉及，包括放大、添加噪音等。
